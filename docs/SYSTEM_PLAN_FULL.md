6. 策略层：先做“自适应网格 + 机器学习过滤”

基础策略：自适应网格

假设你长期持有一只股票，设定一个目标底仓：

base_position=例如总资金的30%−50%

然后根据价格偏离调整仓位：

情况	动作
价格低于VWAP很多，且成交量没有失控	小额买入
价格高于VWAP很多，且可卖库存充足	小额卖出
价格在正常区间	不交易
接近涨停/跌停	停止普通网格逻辑，进入风控模式

核心不是“越跌越买”，而是：

目标仓位=基础仓位+信号强度×调整仓位

例如：

target_position=clip(base+k×signal,min_position,max_position)

其中 clip 是仓位上下限，防止模型越买越多。

机器学习过滤器

模型判断	网格动作

2. 短线信号模型

这个模型每1分钟或5分钟运行一次，回答：

future_net_return_10m = future_return_10m - commission - tax - slippage

模型输出：

P_up = 未来10分钟上涨并覆盖成本的概率
P_down = 未来10分钟下跌并覆盖成本的概率
signal = P_up - P_down

交易含义：

signal	含义	操作
signal > 0.15	短线偏低，可能反弹	允许小额买入
signal < -0.15	短线偏高，可能回落	允许小额卖出
-0.15 到 0.15	没有优势	不交易

四、个股机器学习过程

你的“把它过往的手法、数据走向学到位”可以拆成五个学习任务。

1. 学习价格性格

系统要回答：

这只股票是更容易均值回归，还是更容易单边趋势？

需要学习：

特征	含义
日内高低点分布	容易早盘冲高，还是午后拉升
VWAP偏离后走势	偏离均价后是否容易回归
连续上涨后的表现	涨多后继续涨，还是回落
连续下跌后的表现	跌多后反弹，还是继续跌
跳空后的表现	高开低走、低开高走、还是延续
收盘前行为	尾盘是否容易异动
2. 学习成交性格

系统要知道：

这只股票什么时候容易成交？什么时候成交风险高？

学习内容：

特征	作用
分钟成交量分布	判断哪个时间段流动性最好
成交额分位数	判断今天是否放量或缩量
买卖价差	判断滑点成本
成交冲击	判断下单金额会不会影响价格
盘口厚度	判断限价单成交概率
撤单和未成交比例	判断执行难度
3. 学习市场联动

单只股票不是孤立的。系统要学习它和这些对象的关系：

个股 vs 大盘指数
个股 vs 行业指数
个股 vs 同板块龙头
个股 vs 成交额变化
个股 vs 市场风险偏好

比如：

情况	系统理解
个股跌，但行业没跌	可能是个股弱
行业跌，个股也跌	可能是板块共振
大盘急跌，个股跌	不要轻易低吸
个股逆势放量上涨	可能是强势状态，不宜乱卖
4. 学习交易成本

模型不能只学价格，还要学成本。

每笔交易实际收益应该这样算：

真实收益=价格收益−佣金−印花税−过户费−滑点−冲击成本

财政部、税务总局公告显示，自2023年8月28日起，证券交易印花税实施减半征收；系统里要把税费、佣金、滑点都做成可配置参数，不能写死。

也就是说，模型预测的不是：

未来会不会涨

而是：

未来涨幅能不能覆盖交易成本
5. 学习失败案例

这点很重要。系统不光要学“什么时候赚钱”，更要学：

我以前是怎么亏钱的？

失败案例库可以这样建：

失败类型1：低吸后继续下跌
失败类型2：卖出后继续上涨
失败类型3：信号正确但滑点太大
失败类型4：模型判断正确但风控太松
失败类型5：大盘共振下跌时误判为个股低吸机会
失败类型6：接近跌停时错误加仓
失败类型7：成交额不足导致无法按预期卖出

每天复盘以后，把失败案例写入：

五、机器学习模型结构

不要只做一个大模型。建议拆成四个模型。

模型A：个股画像模型

每天更新一次，判断这只股票今天是否适合交易。

输出：

tradable_score：今日可交易评分
mean_reversion_score：均值回归评分
trend_score：趋势延续评分
liquidity_score：流动性评分
risk_score：风险评分

示例：

if tradable_score < 0.6:
    today_trading_allowed = False
模型B：短线概率模型

每1分钟或5分钟运行一次，输出：

P_up    = 未来N分钟上涨并覆盖成本的概率
P_down  = 未来N分钟下跌并覆盖成本的概率
signal  = P_up - P_down

判断：

signal	解释
> 0.15	有偏多优势
< -0.15	有偏空优势
中间区间	不交易

第一版建议用：

Logistic Regression → LightGBM → XGBoost/CatBoost

不要一开始就上强化学习。强化学习容易在回测里看起来很好，但实盘中非常容易因为市场变化、滑点、成本和样本外失效而崩。

模型C：库存控制模型

这个模型负责决定仓位。

目标仓位=基础仓位+信号强度×最大调整仓位

示例：

基础仓位：40%
最低仓位：10%
最高仓位：60%
单次交易：2%-5%

它的任务不是预测，而是防止系统过度交易、过度补仓、过度集中。

模型D：风控拦截模型

这个模型拥有最高权限。

它检查：

是否超过最大仓位
是否超过单日亏损
是否连续亏损
是否接近涨跌停
是否流动性不足
是否数据异常
是否滑点过大
是否交易次数过多
是否撤单次数过多

上交所《程序化交易管理实施细则》说明，该细则目的包括加强程序化交易监管、促进程序化交易规范发展、维护证券交易秩序和市场公平，并自2025年7月7日起施行。 所以系统要从第一版就设计成低频、限频、可解释、可审计，不要把目标定成毫秒级高频交易。

六、回测系统：没有回测通过，不允许交易

你的原则应该是：

回测不是为了证明策略赚钱，而是为了证明策略没有作弊。

回测必须过三关。

第一关：数据无作弊

检查：

不能用未来数据
不能用未来复权因子错误处理
不能用未来成交量
不能用未来最高价最低价
不能随机切分时间序列
不能只挑赚钱区间
不能忽略停牌、涨跌停和未成交

训练方式必须按时间滚动：

过去数据训练
之后一段验证
再之后一段测试
然后窗口向前滚动

不能这样：

把所有历史数据打乱，然后随机训练测试

因为金融时间序列有强烈时间顺序，随机切分很容易产生虚假的好结果。

第二关：交易制度真实

回测要尽量接近真实交易，至少模拟：

项目	回测要求
交易时间	只在合法交易时间成交
可卖库存	区分昨日可卖库存和今日新增库存
涨跌停	涨停不一定买得到，跌停不一定卖得出
停牌	停牌日不能交易
手续费	每笔扣除
印花税	卖出端计入
滑点	按成交额和盘口模拟
未成交	限价单可能挂单失败
撤单	记录撤单次数和原因
最小交易单位	按市场规则处理
数据延迟	不允许用当时不知道的数据

交易规则会变化，系统要做“规则版本管理”。例如新华网报道，沪深北交易所修订后的交易规则将于2026年7月6日起实施，其中包括盘后固定价格交易适用范围扩展等内容。 这意味着你的系统不能把交易规则永久写死，而要用配置文件管理。

第三关：策略有效

回测通过的最低标准可以设成：

1. 扣除全部成本后仍为正收益
2. 样本外测试没有明显失效
3. 最大回撤在可承受范围内
4. 单笔平均收益大于交易成本
5. 策略收益不是靠少数几笔极端交易撑起来
6. 参数轻微变化后结果不崩
7. 牛市、熊市、震荡市都测试过
8. 模型表现优于无AI的基线网格策略

注意最后一条很重要：

AI必须跑赢简单规则，否则AI没有上线价值。

九、风控系统规划

最大回撤
最大持仓比例
最大连续亏损次数
最大单日交易次数
最大单日撤单次数
最大单笔交易金额
个股级风控
接近跌停禁止加仓
涨停附近禁止追买
成交额不足不交易
波动率异常不交易
盘口价差过大不交易
公告、停牌、异常波动风险期间降低权限
模型级风控
模型置信度不足不交易
模型输出异常不交易
特征缺失不交易
模型版本未审核不交易
今日模型表现明显失真则停机
执行级风控

下单价格检查
下单数量检查
可卖库存检查
重复下单检查
撤单频率检查
成交异常检查

程序化交易系统还要避免异常申报和高频化风险。上交所细则中列出的高频交易情形包括单个账户每秒申报、撤单最高笔数达到300笔以上，或单个账户单日申报、撤单最高笔数达到20000笔以上。 第一版系统应该把交易频率、撤单次数、订单数量都限制在远低于这些边界的位置。

账户总权益=现金+持仓市值

信号持有后最大有利波动
4. 风控复盘

每天检查：

风控是否拦截了错误交易？
有没有该拦截但没拦截？
有没有过度拦截导致错过机会？
有没有接近亏损线？
有没有接近交易次数上限？
有没有接近撤单次数上限？

3.2 短线信号模型：AI只给概率

这个模型每1分钟或5分钟运行一次。

它不输出“买入/卖出”，而是输出：

P_up    = 未来N分钟上涨并覆盖成本的概率
P_down  = 未来N分钟下跌并覆盖成本的概率
signal  = P_up - P_down

标签设计：

future_return_10m = future_price_10m / current_price - 1
future_net_return = future_return_10m - commission - tax - slippage

分类标签：

future_net_return > 正阈值：up
future_net_return < 负阈值：down
中间区域：no_trade

V1模型顺序：

Baseline规则模型
↓
Logistic Regression
↓
LightGBM
↓
XGBoost / CatBoost

V1先不做强化学习。强化学习可以留到研究阶段，但不能作为早期实盘核心。

3.3 库存控制模型：库存控仓位

库存模型的任务：

把AI信号转成目标仓位。

核心公式：

目标仓位=基础仓位+信号强度×最大调整仓位

示例：

基础仓位：40%
最低仓位：10%
最高仓位：60%
单次调整：2%-5%
单日最大买入：总资产10%
单日最大卖出：可卖库存30%-50%

伪代码：

def calculate_target_position(signal, base_pos, min_pos, max_pos, max_adjust):
    raw_target = base_pos + signal * max_adjust
    target = min(max(raw_target, min_pos), max_pos)
    return target

库存模型必须考虑：

当前仓位
可卖库存
今日已买金额
今日已卖金额
今日已实现盈亏
今日浮动盈亏
最大仓位限制
最小仓位限制
连续亏损次数

1. 排除交易风险高的股票

建议先排除：

ST、*ST
退市风险股票
长期停牌或停牌频繁的股票
上市时间太短、历史数据不足的股票
经常涨停跌停的极端情绪股
成交额太低的股票
价差过大、流动性差的股票
近期异常波动过大的股票
重大公告、重组、退市风险期股票

你的系统目标是“长期学习一个股票的性格”，所以不适合选择那种每天靠消息、题材、封板驱动的股票。那类股票模式变化太快，模型容易学到噪音。

2. 排除不适合库存交易的股票

库存型交易需要能买、能卖、能调整仓位，所以不适合：

成交太少
盘口太薄
经常一字板
经常跳空大涨大跌
价格走势长期单边下跌
日内波动太小
日内波动太极端

你的策略不是赌方向，而是利用短期偏离和仓位调节，所以最适合的是：

流动性好、波动适中、走势有一定规律、不是纯消息驱动的股票。

五、每个评分怎么计算
1. 流动性分

看：

20日平均成交额
60日平均成交额
成交额稳定性
分钟成交额分布
早盘、午盘、尾盘成交是否集中
大额成交是否容易冲击价格

初期可以简单设置：

20日平均成交额越高，分数越高
成交额波动越稳定，分数越高
成交极度集中在少数几分钟，扣分

注意：成交额太低的股票，不适合自动化模拟；成交额太高但波动很小的股票，也不一定适合库存策略。

2. 波动空间分

看：

日内振幅
1分钟波动率
5分钟波动率
10分钟波动率
ATR
高低点分布

你要找的不是“波动越大越好”，而是：

波动适中最好。

波动太小，没有利润空间；波动太大，容易触发风控，低吸变接刀。

3. 均值回归分

这是最重要的分数之一。

系统要统计：

价格低于VWAP 1个标准差后，未来10分钟是否反弹？
价格低于VWAP 2个标准差后，未来10分钟是否反弹？
价格高于VWAP 1个标准差后，未来10分钟是否回落？
价格高于VWAP 2个标准差后，未来10分钟是否回落？

示例：

当 z_score < -1.5：
    未来10分钟平均收益 = +0.18%
    胜率 = 58%
    扣成本后收益 = +0.08%

当 z_score > 1.5：
    未来10分钟平均收益 = -0.15%
    胜率 = 55%
    扣成本后收益 = +0.05%

如果扣除成本后仍有正收益，说明这只股票可能适合做库存型高抛低吸。

4. 执行质量分

看：

成交是否连续
价格跳动是否平滑
分钟成交额是否足够
限价单是否容易成交
滑点是否可控
涨跌停附近是否容易失效

第一版没有盘口数据也没关系，可以先用1分钟K线近似估计：

成交额越高，执行质量越好
分钟振幅过大，执行质量扣分
经常跳空或快速拉跌，执行质量扣分

以后有盘口数据，再加入：

买卖价差
盘口厚度
委买委卖不平衡
撤单率
冲击成本
5. 稳定性分

看这只股票的行为是不是经常变化。

统计：

过去一年和最近三个月的波动特征是否一致
均值回归规律是否稳定
成交额是否突然变差
交易机会是否集中在少数极端日
回测收益是否由少数几天贡献

如果一个股票大部分收益来自一两天暴涨暴跌，不适合作为长期学习对象。

6. 风险惩罚分

扣分项包括：

涨跌停次数多
跌停附近成交困难
停牌或异常公告频繁
跳空幅度大
单边下跌趋势明显
成交额快速萎缩
大盘下跌时 Beta 过高
板块风险过高

这部分要宁可严格一点。选股阶段的任务不是找到最刺激的股票，而是找到最适合被系统稳定学习的股票。

十一、每日复盘后的调整规则

调整不能随便拍脑袋，要有规则。

1. 股票池调整
连续3天模拟表现差 → 从模拟池降级到学习池
连续5天数据异常 → 降级到观察池
连续10天表现稳定 → 升级为主跟踪候选
出现重大异常 → 直接暂停
2. 信号阈值调整
假信号过多 → 提高买卖阈值
机会错过过多 → 轻微降低阈值
高置信度仍亏损 → 模型重训
低置信度却盈利多 → 检查标签设计
3. 仓位参数调整
亏损主要来自加仓太快 → 降低单次加仓比例
亏损主要来自仓位太高 → 降低最大仓位
盈利主要来自小仓位试错 → 保持低仓位策略
卖飞太多 → 检查卖出阈值，不直接放宽风控
4. 风控规则调整
跌停附近亏损多 → 扩大跌停保护区
大盘共振下跌亏损多 → 增加指数过滤
板块共振下跌亏损多 → 增加行业过滤
成交额萎缩导致亏损 → 提高流动性门槛

八、一周观察期的复盘内容

每天复盘不用太复杂，先围绕五个问题：

1. 哪些股票稳定排在前面？

说明它们不是单日偶然入选。

2. 哪些股票排名波动很大？

这类股票可能不稳定，不适合作为深度学习对象。

3. 哪些股票流动性突然变差？

流动性恶化要降级。

4. 哪些股票波动太极端？

极端波动可能是题材、消息、异常资金推动，不一定适合库存模型。

5. 哪些股票出现涨跌停、跳空、异常事件？

这类股票要降低优先级，甚至暂停观察。

class SelectionEngine:

    def __init__(self, config, pool_manager, scoring_pipeline, report_system):
        self.config = config
        self.pool_manager = pool_manager
        self.scoring_pipeline = scoring_pipeline
        self.report_system = report_system

    def run(self, stock_universe, market_data):
        """
        主流程入口
        """

        # Step 1: Hard Filter
        filtered_stocks = self._apply_hard_filter(stock_universe, market_data)

        # Step 2: Feature + Score
        scored_stocks = self.scoring_pipeline.run(filtered_stocks, market_data)

        # Step 3: Rank
        ranked = sorted(scored_stocks, key=lambda x: x["total_score"], reverse=True)

        # Step 4: Pool update
        pools = self.pool_manager.update(ranked)

        # Step 5: Report
        self.report_system.generate(ranked, pools)

        return pools

    def _apply_hard_filter(self, stocks, data):
        from stock_selection.hard_filter import HardFilter
        return HardFilter().filter(stocks, data)

class ScoringPipeline:

    def __init__(self):
        from stock_selection.liquidity_score import LiquidityScore
        from stock_selection.volatility_score import VolatilityScore
        from stock_selection.mean_reversion_score import MeanReversionScore
        from stock_selection.execution_score import ExecutionScore
        from stock_selection.risk_penalty import RiskPenalty

        self.modules = [
            LiquidityScore(),
            VolatilityScore(),
            MeanReversionScore(),
            ExecutionScore(),
            RiskPenalty()
        ]

    def run(self, stocks, market_data):

        results = []

        for stock in stocks:
            score = {
                "stock_code": stock,
                "liquidity": 0,
                "volatility": 0,
                "mean_reversion": 0,
                "execution": 0,
                "risk_penalty": 0,
                "total_score": 0
            }

            for module in self.modules:
                module_score = module.calculate(stock, market_data)
                score.update(module_score)

            # 总分计算
            score["total_score"] = (
                score["liquidity"] * 0.3 +
                score["volatility"] * 0.25 +
                score["mean_reversion"] * 0.2 +
                score["execution"] * 0.15 -
                score["risk_penalty"]
            )

            results.append(score)

        return results
3️⃣ pool_manager.py（状态机核心）
class PoolManager:

    def __init__(self):
        self.state = {}  # stock_code -> state

    def update(self, ranked_list):
        """
        更新股票池状态
        """

        watch_pool = []
        learning_pool = []
        focus_pool = []

        for i, stock in enumerate(ranked_list):

            code = stock["stock_code"]
            score = stock["total_score"]

            # 状态机逻辑
            if score < 0.3:
                self.state[code] = "REJECTED"

            elif score < 0.5:
                self.state[code] = "WATCH"
                watch_pool.append(code)

            elif score < 0.7:
                self.state[code] = "LEARNING"
                learning_pool.append(code)

            else:
                self.state[code] = "FOCUS"
                focus_pool.append(code)

        return {
            "watch_pool": watch_pool[:100],
            "learning_pool": learning_pool[:20],
            "focus_pool": focus_pool[:5]
        }

    def __init__(self, engine, data_loader):
        self.engine = engine
        self.data_loader = data_loader

    def run_daily(self, date):

        # Step 1: load universe
        stock_universe = self.data_loader.load_universe()

        # Step 2: load market data
        market_data = self.data_loader.load_market_data(date)

        # Step 3: run engine

        return pools

class LiquidityScore:

    def calculate(self, stock, data):
        avg_amount = data[stock]["avg_amount_20d"]

        score = min(avg_amount / 1e8, 1.0)

        return {"liquidity": score}

class VolatilityScore:

    def calculate(self, stock, data):
        vol = data[stock]["intraday_volatility"]

        score = min(vol / 0.05, 1.0)

        return {"volatility": score}

class MeanReversionScore:

class RiskPenalty:

    def calculate(self, stock, data):

        penalty = 0

        if data[stock]["limit_up_days"] > 3:
            penalty += 0.3

        if data[stock]["suspension_risk"]:
            penalty += 0.5

        return {"risk_penalty": penalty}
